## ❓ WHY SKAFFOLD + MONOREPO RESEARCH?

### **The Problem:**

```
You have 5 repos:
1. agent-town/ (Next.js + Phaser)
2. openclaw-office/ (Vite + R3F)
3. ai-town/ (Convex + PixiJS)
4. Star-Office-UI/ (Flask + Python)
5. landing-page/ (Vanilla HTML/CSS/JS)

Problem:
❌ Need 5 terminals to run all
❌ No unified deployment
❌ No hot-reload across repos
❌ Manual setup for each
❌ No Kubernetes configuration
```

### **The Research:**

**Question:** "How do we unify 5 separate repos for development?"

**Options Researched:**
1. **Monorepo** - Merge all into 1 repo
2. **Multi-Repo + Skaffold** - Keep separate, unify deployment
3. **Docker Compose** - Local orchestration
4. **DevContainers** - Standardized environments

**Decision:** ✅ **Multi-Repo + Skaffold**

**Why:**
```
✅ Preserves repo independence (can develop separately)
✅ Enables coordination (unified deployment)
✅ AI-friendly (Nx provides context)
✅ Skaffold handles Kubernetes deployment
✅ Hot-reload across all 5 projects
✅ 1 terminal instead of 5
```

### **What Was Implemented:**

```
✅ Root skaffold.yaml (orchestrates all 5)
✅ Individual skaffold.yaml per repo
✅ Dockerfile per repo
✅ Kubernetes manifests per repo
✅ Ready for: skaffold dev (one command)
```

### **Benefits:**

```
BEFORE:
❌ 5 terminals
❌ Manual coordination
❌ No K8s deployment
❌ No hot-reload

AFTER:
✅ 1 terminal (skaffold dev)
✅ Auto-deploy to Kubernetes
✅ Hot-reload all 5 projects
✅ Production-parity environment
```
