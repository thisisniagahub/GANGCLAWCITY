## 🎯 File Organization Principles

### **What Goes Where**

| File Type | Location | Example |
|-----------|----------|---------|
| User-facing docs | `/docs/` | Guides, tutorials, APIs |
| Technical specs | `/docs/01-architecture/` | Architecture, design docs |
| Deployment guides | `/docs/02-deployment/` | Deploy instructions |
| Config files | `/config/` | YAML, JSON configs |
| Scripts | `/scripts/` | .bat, .sh, .js scripts |
| Security docs | `/security/` | Audits, policies |
| Code | `/apps/*/` | Application code |
| Master context | Root (`QWEN.md`) | Critical project context |

### **What Stays in Root**
- ✅ `QWEN.md` - Critical master context
- ✅ `package.json` - If used for root-level scripts
- ✅ `.gitignore` - Git configuration
- ✅ `LICENSE` - Legal requirement
- ✅ `README.md` - Optional (moved to `/docs/00-start-here/`)
