**Coding Standards:**
```typescript
// ✅ GOOD: Meaningful names, small functions
function calculateElapsedTimeInDays(startDate: Date, endDate: Date): number {
  const millisecondsPerDay = 1000 * 60 * 60 * 24;
  return (endDate.getTime() - startDate.getTime()) / millisecondsPerDay;
}

// ❌ BAD: Unclear names, large functions
function calc(d1: Date, d2: Date): number {
  // ... 100 lines of code
}
```

**Best Practices:**
- ✅ **Small commits** (<200 lines per commit)
- ✅ **Feature branches** (feature/xxx naming)
- ✅ **PR within 24 hours** (max 400 lines)
- ✅ **Unit tests** (80%+ coverage)
- ✅ **TypeScript strict mode** (no `any`)
