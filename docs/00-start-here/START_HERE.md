# GANGCLAWCITY Start Here Checklist

Use this file when you first enter the repository or when you need to reset context quickly.

## 1. Understand The Repo Shape

- This is a workspace of multiple apps.
- Two folders are current product priorities: `openclaw-office/` and `agent-town/`.
- Three folders are donor modules under evaluation: `ai-town/`, `Star-Office-UI/`, and `pixel-agents/`.

## 2. Read The Active Docs In Order

1. `README.md`
2. `docs/DOCS_INDEX.md`
3. `docs/01-architecture/ARCHITECTURE.md`
4. `docs/02-deployment/DEPLOYMENT_STATUS.md`

## 3. Pick The Right Workstream

- Working on dashboard and control UX: use `openclaw-office/`
- Working on spatial demo and operator interactions: use `agent-town/`
- Working on R&D or donor assessment: use the donor folders directly, but do not present them as finished GANGCLAWCITY products

## 4. Follow The Safety Rules

- Do not expose production gateway tokens to browser clients.
- Do not trust historical deployment claims without re-checking them.
- Do not remove upstream license files when rebranding product surfaces.

## 5. Use Realistic Commands

See `docs/00-start-here/QUICKSTART.md` for commands that match the current tree.
